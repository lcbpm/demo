### 面试题

```text
1.Redis中多大的Key算热key,该如何解决?
热key是指在一段时间内访问频率非常高的key。解决热key问题的方法包括：
分片：将热key分散到多个Redis实例中。
缓存预热：在高峰期前预先加载热key。
限流：对访问频率进行限制，防止单个key被频繁访问。

2.如果让你设计一个分布式链路跟踪系统?你怎么做?
可以使用开源工具如Zipkin或Jaeger，结合日志收集和分析系统。关键步骤包括：
生成唯一的追踪ID：每个请求都有唯一的追踪ID。
日志收集：在每个服务节点上记录日志，并附上追踪ID。
数据存储和分析：将日志数据存储在集中式存储系统中，进行分析和可视化。

3. 如何用Redis统计亿级网站的UV?
使用HyperLogLog数据结构，它可以在固定内存空间内提供近似去重计数。具体步骤：
每当用户访问时，将用户的唯一标识符添加到HyperLogLog中。
使用PFCOUNT命令统计UV。

4. 如果要你实现Spring IOC,你会注意哪些问题?
关注以下几个方面：
依赖注入：确保Bean的依赖关系正确注入。
Bean生命周期管理：管理Bean的创建、初始化、销毁等生命周期。
配置和注解：合理使用XML配置和注解，避免冲突。

5.Spring Boot的约定优于配置,你的理解是什么?
Spring Boot通过默认配置简化开发，开发者只需在必要时进行配置覆盖。具体体现包括：
自动配置：Spring Boot会根据类路径中的依赖自动配置应用。
约定目录结构：如src/main/resources下的application.properties文件。

6.Spring 是如何管理事务的,事务管理机制?
Spring通过AOP（面向切面编程）实现事务管理，使用@Transactional注解定义事务边界。事务管理机制包括：
事务传播行为：定义事务如何在多个方法间传播。
事务隔离级别：控制事务间的隔离程度。

7.k8s中pod和node是什么关系?
Pod是Kubernetes中最小的部署单元，Node是运行Pod的物理或虚拟机。Pod内的容器共享网络和存储资源，Node负责执行和管理Pod。

8. 如果让你设计一个推荐系统,你会从哪些纬度来考虑?
设计推荐系统时需要考虑以下纬度：
用户行为：分析用户的点击、浏览、购买等行为。
内容特征：提取物品的特征，如类别、标签等。
协同过滤：基于用户相似性或物品相似性进行推荐。
实时性：确保推荐结果的实时更新和响应。

9.Mysql的行级锁到底锁的是什么东西?
行级锁锁定的是表中的某一行数据，确保事务的隔离性。具体实现包括：
共享锁和排他锁：分别用于读操作和写操作。
间隙锁：锁定索引键之间的空隙，防止幻读。

10. 为什么k8s的设计里有pod,而不是直接用容器?
Pod可以包含多个容器，提供更高的灵活性和管理能力。Pod中的容器共享网络和存储资源，简化了容器间的通信和资源管理。

11.CPU 使用率较高排查和解决思路?
检查系统负载、进程占用情况，优化代码和查询，增加硬件资源。

12. 什么东西可以当做 GC Root,跨代引用怎么办?
GC Root包括栈中的局部变量、静态变量等。跨代引用可以通过引用计数或弱引用来处理。
```
