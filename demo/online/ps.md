### ps -M -p <PID>

```text
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1 103332  8688 ?        Ss   08:20   0:02 /sbin/init
root         2  0.0  0.0      0     0 ?        S    08:20   0:00 [kthreadd]
user      1234  1.2  1.5 105432 31876 pts/0    Sl   09:00   0:34 /usr/bin/python3
user      5678 10.0  2.0 265728 60484 pts/1    R    09:02   1:12 /usr/bin/java

PID: 进程ID。唯一标识系统中的每个进程。
LWP: 轻量级进程（线程）ID。表示每个线程的唯一ID。
TT: 控制终端。显示进程关联的终端，如果显示??，表示没有关联任何终端。
STAT: 进程状态。以下是一些常见的状态：
R: 运行
S: 睡眠
D: 不可中断的睡眠
T: 停止
Z: 僵尸进程
I: 空闲内核线程
TIME: 使用的累计CPU时间。表示进程运行消耗的CPU时间。
COMMAND: 执行的命令。表示启动进程的命令及其参数。
%CPU: CPU使用率。表示进程占用的CPU百分比。
%MEM: 内存使用率。表示进程占用的物理内存百分比。
VSZ: 虚拟内存大小。进程使用的虚拟内存量，以KB为单位。
RSS: 常驻内存大小。进程使用的物理内存量，以KB为单位。
STARTED: 启动时间。表示进程的启动时间。

CPU：显示所有CPU核心的使用情况，图形表示每个核心的负载。
Mem：显示内存的使用情况，包括已用、空闲、缓存和缓冲区的内存。
Swp：显示交换分区的使用情况，包括已用和空闲的交换空间。

PID：进程ID，系统中唯一标识每个进程的编号。
USER：进程所属的用户。
PRI：进程的优先级，值越低，优先级越高。
NI：进程的nice值，用于调整进程的优先级。
VIRT：进程使用的虚拟内存大小，包括所有已分配的内存。
RES：进程使用的常驻内存大小，即实际使用的物理内存。
SHR：进程使用的共享内存大小。
S：进程的状态。常见状态包括：
R：运行中
S：睡眠中
D：不可中断的睡眠中
T：停止状态
Z：僵尸进程
%CPU：进程占用的CPU百分比。
%MEM：进程占用的物理内存百分比。
TIME+：进程累积的CPU时间，包括用户态和内核态的时间。
COMMAND：进程的启动命令和参数。

```
